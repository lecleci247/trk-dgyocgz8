Marked-but-correct (false positives)

UWorld's Marked and Incorrect filters overlap — a question can be both. To avoid double-counting the redo pool, get this number by selecting Correct + Marked together in UWorld (reads as a union), then: Correct + Marked − that union = marked-but-correct.

Current miss rate (auto) —

Miss rate = incorrect ÷ used, where used = total questions − unused remaining. This projects how many of your remaining unused questions will likely need a redo pass too — not just what's already known-incorrect today. Only change "Total questions" if UWorld adds content to the bank.

iOS Health Shortcut
Build this once. After that, opening the app and tapping "Run PPLUA Sync" auto-fills sleep, resting HR, steps, and water. WaterMinder writes water into Apple Health, so it imports too.
Build the shortcut
Open Shortcuts → tap + → rename the shortcut to exactly PPLUA Sync (must match exactly — this is how the app's sync button finds it).
Add Find Health Samples → Sample Type: Sleep Analysis → Sort by: Start Date → Order: newest first → Limit: On, Get 1.
Add Get Details of Health Sample twice on that result — once set to Start Date, once to End Date. Then add Get Time Between Dates using those two outputs, Unit: Hours. That's your sleep hours.
Add Find Health Samples → Resting Heart Rate → newest first → Limit 1. Add Get Details of Health Sample → Value.
Add Find Health Samples → Step Count → Add Filter: Start Date is Today → no limit. Add Calculate Statistics → Sum. That's today's steps.
Add Find Health Samples → Dietary Water → Add Filter: Start Date is Today → no limit. Add Calculate Statistics → Sum. Add Calculate to divide by 950 for bottles (check units the first run — see full walkthrough in chat).
Add Find Health Samples → Body Mass → newest first → Limit 1. Add Get Details of Health Sample → Value. No need to gate this by day — the app only applies it on Mondays automatically.
Final action: Open URLs. Build the URL below directly in that field, inserting each variable chip where shown.
Getting the exact URL

Once this app is hosted, open it on your phone and come back to this Setup tab — the code block above will show your real, live URL with the correct domain already filled in. Copy that instead of retyping it, to avoid a typo breaking the sync.

Why the tap is needed

Apple Health has no web connection, so a web app can't read it silently. This Shortcut is the bridge: it reads Health and hands the numbers to the app through the link. One tap when you open the app in the morning covers the whole day. Everything else in here runs automatically from that data.

Host it for full auto-fill

Right now this runs from a saved file. If you host it on GitHub Pages (free), the Shortcut can open a real URL and the values land without you pasting anything. Ask in the project chat and I'll walk you through the 10-minute setup.

Full data export (backup / Excel)
Downloads every day you've ever logged as a CSV file — dates, checklist completion, sleep, HR, water, UWorld/Anki, and every timestamp. This is your durable backup: opens in Excel, Numbers, or Google Sheets, and it's the fastest way to hand months of history to Claude for deeper analysis.
Downloads to Files app or your browser's downloads.
Weekly review export
One tap turns your last 7 logged days into plain text. Paste it to Claude for trend analysis, adherence review, and deload or program decisions the app's automatic triggers can't make on their own.
